How to open a relay?


You need this for it to work:
  • 1. Windows Server, atleast 1GB RAM for correct work.
  • 2. Static IP, you can use NOIP or another dynamic DNS.
  • 3. Google DNS
  • 4. mouseTor Certificate (Instruction on how to get one will be later on this page)

Part 1: Windows Server.

For correct work you need to have a Windows Server OS

If you have this, part 1 is complete.

Part 2: Static IP, Dynamic DNS.

If you have any static IPs (non local, but in local network it will work.)

or you have connected Dynamic DNS: After connecting DDNS/Static IP you will need to configure Virtual Servers.

You need to open Port 80 for your server or Port 443 for SSL

At Windows Server you need to configure the Firewall to accept connections to Port 80.

After this, you can start your own mouseTor Website relay. You need to start NGINX/APACHE for this.

Part 3: Google DNS

Google DNS Servers are required for a server if you want to have high speed connections and fast MX/NS Records update.

4. mouseTor Certificate

If you have started a relay and everything is okay, you need configure some files.

  • Create a file certificate.ttp (For your certificate at the server)
  • Create a file config.dns (For relay to correctly work.)

Write to config.dns your first record

test.monion [SERVER IP]

 and if you have more files, then add index.html or another file.

test.monion/filename [SERVER IP]/filename

For the certificate you need to have a base64 record, create it at base64encode.net

How to create a correct certificate?

You need to encode text to base64, text needs to look like this AU29AMFGI99A99209AJFMG89AM29

For example: AU29AMFGI99A99209AJFMG89AM29 -> QVUyOUFNRkdJOTlBOTkyMDlBSkZNRzg5QU0yOQ== (VALID!)

After Mouse has added your certificate to the valid list at official certificate center, you can add the record to your relay. Write the valid record to certificate.ttp

In config.dns you can add any count of records, 1 million or 10 millions, no limits, but if you have an empty config.dns file, all sites will be unavailable from mouseTor Client.
