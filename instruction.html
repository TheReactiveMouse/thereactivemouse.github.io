h1>How to open relay?
You need have to do this successfully :
  • 1. Windows Server, minimal 1GB RAM For correct work.
  • 2. Static IP, allowed using NOIP or another dynamic DNS.
  • 3. Google DNS
  • 4. mouseTor Certificate (How to made i will be show in instruction.)

Part 1, Windows Server.

For correct work you need have a Windows Server

If you have this, part 1 is complete.

Part 2, Static IP, Dynamic DNS.

If you have an any static ip(non local, but in local network it's will work.)

or you connected Dynamic DNS.After connecting DDNS/Static IP you need configure Virtual Servers.

You need open for your server port 80(HTTP) or 443 for SSL

At Windows Server you need configure Firewall to accept 80 port connections.

After this, you can start your own mouseTor Web Site relay.You need start NGINX/APACHE for this.

Part 3, Google DNS

Google DNS Servers is required for server, if you want have a high speed of connections and fast MX/NS Records update.

4. mouseTor Certificate

If you started relay and all okay, you need configure files.

  • Create file certificate.ttp(For your certificate at server.)
  • Create file config.dns(For relay correctly work.)

Write to config.dns your first record

test.monion *server ip.*

 and if you have a more files then index.html or another file.

test.monion/filename *server ip/filename

For certificate you need have a base64 record, create it at site base64encode.net

How to create a correct certificate?

You need encode text to base64, text need look like this AU29AMFGI99A99209AJFMG89AM29

For example AU29AMFGI99A99209AJFMG89AM29 -> QVUyOUFNRkdJOTlBOTkyMDlBSkZNRzg5QU0yOQ== (VALID!)

After mouse added your certificate to valid at official certificate center, you can add record to your relay.Write valid record to certificate.ttp

In config.dns you can add any count of records, 1 million or 10 millions, no limits, if you will have a empty config.dns file, all sites will unavailable from mouseTor Client.
